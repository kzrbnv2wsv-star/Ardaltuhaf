# كتالوج أرض التحف للتجارة — دليل التشغيل والنشر

كتالوج جملة إلكتروني (B2B) يعمل بالكامل في المتصفح بدون خادم. مناسب للنشر المجاني على GitHub Pages.

## الملفات

| الملف | الوصف |
|------|-------|
| `index.html` | واجهة العميل: عرض المنتجات، البحث، الفلترة، السلة، الطلب عبر واتساب |
| `admin.html` | لوحة الإدارة: إضافة/تعديل/حذف المنتجات، وتصدير ملف البيانات |
| `products.json` | بيانات المنتجات وبيانات الشركة (المصدر الوحيد للحقيقة) |

---

## أولاً: الإعداد قبل النشر

1. افتح `admin.html` في المتصفح (دبل كليك).
2. من قسم **«بيانات الشركة والإعدادات»**: أدخل **رقم واتساب** بالصيغة الدولية بدون `+` أو صفر البداية (مثال: `966551234567`).
3. أضف منتجاتك الفعلية من زر **«＋ منتج جديد»**.
4. اضغط **«⬇ تصدير products.json»** — سيُحمّل ملف جديد.
5. استبدل ملف `products.json` القديم بالملف الجديد.

> لتعديل بيانات موجودة لاحقاً: افتح `admin.html`، اضغط **«📁 استيراد ملف»** وحمّل نسخة `products.json` الحالية، عدّل، ثم صدّر من جديد.

---

## ثانياً: النشر على GitHub Pages

1. أنشئ حساباً على github.com (إن لم يكن لديك).
2. أنشئ مستودعاً جديداً (Repository) باسم مثل `ardaltuhaf-catalog`.
3. ارفع الملفات الثلاثة: `index.html` و `admin.html` و `products.json`.
4. من إعدادات المستودع: **Settings → Pages**.
5. تحت **Source** اختر الفرع `main` والمجلد `/ (root)`، ثم **Save**.
6. بعد دقائق سيكون الرابط جاهزاً: `https://USERNAME.github.io/ardaltuhaf-catalog/`

---

## إضافة صور المنتجات

1. أنشئ مجلداً باسم `images` داخل المستودع.
2. ارفع صور المنتجات إليه.
3. في لوحة الإدارة، ضع في حقل **«رابط الصورة»**: `images/اسم-الصورة.jpg`
4. صدّر `products.json` وارفعه.

إن تركت حقل الصورة فارغاً، يُعرض رمز بدلاً منها.

## ما الذي يعمل الآن (بدون خادم)

✅ صفحة رئيسية + نبذة + تواصل + زر واتساب
✅ منتجات بسعر الدرزن والكرتون وشدّ الكرتون وحالة التوفر
✅ بحث بالاسم/الكود + فلترة بالتصنيف + ترتيب (جديد/الأكثر/السعر)
✅ شارات «جديد» و«الأكثر طلباً»
✅ تفاصيل المنتج والمواصفات
✅ سلة طلب كاملة (درزن/كرتون، كميات، إجمالي)
✅ نموذج تأكيد (اسم/جوال إجباري/شركة/مدينة/ملاحظات)
✅ إرسال طلب منسّق تلقائياً إلى واتساب
✅ لوحة إدارة مبسّطة للمنتجات
✅ متجاوب 100% مع الجوال

## ما يحتاج مرحلة ثانية (خادم + قاعدة بيانات)

⏳ حفظ تعديلات الإدارة تلقائياً (بدل التصدير اليدوي)
⏳ رفع الصور والفيديو من داخل اللوحة
⏳ تقارير وإحصائيات (الزوار، الأكثر مشاهدة، معدل التحويل)
⏳ إشعارات المنتجات الجديدة

تُبنى هذي باستخدام خدمة مثل Supabase أو Firebase في مشروع منفصل.
